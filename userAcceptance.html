User Acceptance Plan

User Acceptance Plan is a type of testing performed by the end user or the client to verify/accept the software system before moving the software application to the production environment. UAT is done in the final phase of testing after unit, integration and system testing is done. User Acceptance Testing is essential because developers might have built software based on requirements by their own understanding, so for testing whether the final product is accepted by client/end-user, user acceptance testing is needed.

An early version of Parley web app will be provided to Namukolo to test the performance of the app based on scripted test cases.

1 Web App Design Overall design should be clear, consistent and responsive throughout user’s experience

5 Test App Meaningful design of App page that assists in user’s understanding
6 Test App Cookies should remember the session

8 Test Questions Question index and score should be visible during the playing session and should update properly
9 Test Questions Buttons should work properly without lugging and their existence should be meaningful (e.g. skip button should appear only when a question can be skipped)
10 Test Questions Answer type must correspond to question
11 Test Questions Working QR code and functional switch button
12 Test Leaderboard Buttons/icons should work as expected (home-icon, play again, refresh)
13 Test Leaderboard Current user’s score and ranking is clearly visible and distinguishable from the leaderboard
14 Test Leaderboard Leaderboard should properly list the teams in a sorted manner
